实用工具介绍：waifu2x

发表于 2022-02-20 分类于 实用工具

背景

大家在上网冲浪的过程中肯定时不时会碰到自己喜欢的图片，想将它设置为头像或者壁纸。但是美中不足的是，下载的大多数图片要么大小过小，要么噪点过多，这就导致图片经过拉伸后糊成一片，极其不美观。

有什么工具能对付这种尴尬的情况呢？

介绍

waifu2x是由日本程序员nagadomi开发的一个工具程序，它的主要功能是使用“深度卷积神经网络”对输入的动漫风格图像进行放大去噪处理，生成的图像不仅分辨率提高了，噪点也显著减少了。程序使用Lua语言编写，在Github上开源，只能在Linux平台上运行。作者同时还搭建了一个demo网站用于提供在线转换功能。

使用方法

在线版

网站的界面非常傻瓜化，在“选择图像”中上传图像，设置好“降噪程度”和“放大倍数”，通过reCAPTCHA验证后就可以点击“下载”按钮下载结果了。需要注意的一点是由于国内网络环境问题，reCAPTCHA验证有可能无法加载，这就需要自行百度来解决了，例如下面这个链接指向的文章就提供了一种解决方案。

reCAPTCHA无法加载的解决方案

离线版

在线网站虽然容易操作，但是其对文件大小、图像尺寸均作出了限制。因此，倘若在线网站无法满足你的需求，你就可以在本地计算机上运行waifu2x程序以无限制地对图像进行处理。考虑到绝大多数用户所使用的操作系统是Windows，这里只介绍在Windows上如何下载运行waifu2x。

waifu2x-caffe

虽然作者没有把waifu2x移植到Windows平台上，但是Github上有几位程序员以各种方式在Windows平台上重新实现了waifu2x，下面介绍的是由日本程序员lltcggie重新实现的waifu2x程序，叫做waifu2x-caffe。它使用C++编写，借助了一个名为caffe的深度学习框架，只能使用NVIDIA的GPU进行加速处理。程序还提供了一个中文的GUI界面，易于操作。

Github下载地址

下载解压后，双击“waifu2x-caffe.exe”即可打开下面这个GUI界面。

先在“输入和输出设置”中设置好“输入路径”、“输出路径”和“输出文件格式”，接着在“转换质量和处理设置”中设置好降噪和放大的相关选项，“模型”下拉框保持默认即可。接着，点击“应用设置…”，弹出下面这个对话框。

这里只要确保“处理器”选项选择“CUDA”即可。倘若选择“CPU”，程序就会使用CPU来处理图像，这会使处理速度极其缓慢。你还要确保计算机里安装的是NVIDIA的GPU，否则程序将无法正常运行。

点击“确定”按钮后，点击“开始”按钮即可开始处理图像。耐心等待一段时间，如果不出意外，界面下方的文本框中将显示“转换成功”，处理完成的图像将被放入输出路径。

waifu2x-ncnn-Vulkan（对命令行不熟悉的读者可以跳过这一节）

waifu2x-caffe之所以只能在装有NVIDIA的GPU的计算机上运行，是因为其调用了NVIDIA提供的CUDA API进行加速。不得不说，这对Intel/AMD的GPU是不公平的。好在，Github上一位叫nihui的程序员也重新实现了waifu2x程序，叫做waifu2x-ncnn-Vulkan。它也使用C++编写，借助了腾讯的ncnn神经网络计算框架。因为程序调用了所有品牌的GPU都支持的Vulkan API进行加速，所以它可以在装有任意GPU的计算机上运行。不过，因为程序只提供了CLI界面，所以它的使用方法会略微复杂一点。

Github下载地址

下载解压后，在解压目录下启动一个Shell（命令行解释器），输入“waifu2x-ncnn-vulkan -h”可以得到帮助信息，我简单把重要的信息翻译一下。

用法: waifu2x-ncnn-vulkan -i infile -o outfile [options]…

-h 显示这段帮助信息
-v 详细输出
-i input-path 输入图像文件名 (jpg/png/webp) 或路径
-o output-path 输出图像文件名 (jpg/png/webp) 或路径
-n noise-level 降噪等级 (-1/0/1/2/3, default=0)
-s scale 放大比例 (1/2/4/8/16/32, default=2)
-t tile-size tile size (>=32/0=auto, default=0) can be 0,0,0 for multi-gpu
-m model-path waifu2x 模型路径 (default=models-cunet)
-g gpu-id gpu device to use (-1=cpu, default=auto) can be 0,1,2 for multi-gpu
-j load:proc:save thread count for load/proc/save (default=1:2:2) can be 1:2,2,2:2 for multi-gpu
-x enable tta mode
-f format 输出图像格式 (jpg/png/webp, default=ext/png)
